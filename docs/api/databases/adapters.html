数据库适配器¶

Feathers数据库适配器是提供 services 的模块, 它使用 通用API 为特定数据库实现标准 CRUD 功能, 用于初始化和设置, 并提供 查询.

重要

服务 允许实现对 任何 数据库的访问, 这里列出的数据库适配器只是带有通用API的便捷包装器.您仍然可以为此处未列出的数据库获取Feathers功能.另请参阅 社区数据库适配器 列表

支持以下数据库:

内存/文件系统¶

  • feathers-memory - 内存数据库适配器

  • feathers-localstorage - 适用于 验证客户端 的适配器, 可以使用浏览器 LocalStorage 或 ReactNative的 AsyncStorage.

  • feathers-nedb - 用于 NeDB 基于内存或文件系统的独立数据库的数据库适配器.

SQL¶

  • feathers-knex - 适用于 KnexJS 的适配器, 是支持PostgreSQL, MySQL, SQLite和MSSQL的NodeJS的SQL查询构建器

  • feathers-sequelize - 适用于 Sequelize 一个支持PostgreSQL, MySQL, SQLite和MSSQL的NodeJS的ORM的适配器

  • feathers-objection - 用于 Objection.js的服务适配器 - 在Knex之上构建的最小SQL ORM.

NoSQL¶

  • feathers-mongoose - 一个数据库适配器, 用于 Mongoose 一个NodeJS和MongoDB的对象建模库

  • feathers-mongodb - 使用官方NodeJS数据库驱动程序的 MongoDB 的数据库适配器

  • feathers-elasticsearch - 用于 Elasticsearch 的数据库适配器

  • feathers-rethinkdb - 用于 RethinkDB 实时数据库的数据库适配器.
